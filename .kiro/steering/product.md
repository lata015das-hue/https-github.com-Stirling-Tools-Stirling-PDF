# Product context — Stirling-PDF Free Frontend

## What it is
A zero-build, standalone frontend that surfaces every **free** feature of
Stirling-PDF (~56 tools across 5 categories) and talks to a self-hosted
Stirling-PDF backend at `http://localhost:8080` by default.

## Audience
- Privacy-conscious users who want browser-based PDF tools without uploading to
  third-party SaaS.
- Self-hosters and homelab users.
- Devs evaluating an open-source alternative to Adobe Acrobat / iLovePDF / Smallpdf.

## Categories (canonical)
- `page-operations` — 13 tools (merge, split, rotate, crop, page numbers, …)
- `convert` — 15 tools (PDF↔Word/Image/Text/HTML/CSV/Markdown/EPUB/PPT/…)
- `security` — 11 tools (password, watermark, redact, sign, sanitize, …)
- `other` — 11 tools (OCR, extract images, flatten, metadata, stamps, …)
- `advance` — 6 tools (compress, repair, scanner effect, invert, …)

Source of truth for tool ids/names/categories: `js/tools-data.js` (`TOOLS` array).

## Positioning vs paid tier
Free tier = everything in `TOOLS`. The licensed/paid Stirling-PDF Server tier
adds team management, SSO, audit logging, usage analytics, AI features, user
management, and custom branding — these are **out of scope** for this frontend.

## SEO posture
- Primary intent buckets: transactional ("merge pdf", "compress pdf"),
  informational ("how to make a scanned pdf searchable"), commercial
  ("best free pdf merger", "stirling pdf vs ilovepdf"), and questions (PAA).
- Differentiators to lean on in copy: free, no signup, self-hostable, open
  source, privacy (files stay on your infra).
- Languages with first-class OCR support today: en, ar, fr, de, es, zh-Hans.

## Routing reality (today vs target)
- Today: hash-based SPA (`index.html#/tool/<id>`) — not crawlable.
- Target: static pages under `src/routes/tools/<slug>.tsx` (or `/pages/<category>/<slug>.html`)
  with canonical metadata, that link into the SPA for the actual tool UI.
  Sitemap and structured data are generated by Kiro hooks
  (`schema-generator`, `sitemap-updater`, `mark-article-published`,
  `optimize-article`).

## Keep this in mind for any content/SEO task
1. Map every keyword to a real tool id in `js/tools-data.js`. Never invent ids.
2. Default link target for a tool today: `/index.html#/tool/<id>`. Once static
   pages exist, prefer the static path and treat the SPA URL as a redirect target.
3. Don't fabricate prices, ratings, authors, or dates in metadata or schema.
4. The free tier is genuinely free — pricing schema, when used, is `price: "0"`,
   `priceCurrency: "USD"` only when the page itself states "free".
